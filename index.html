プライバシーポリシー — リアルタイム文字起こし

最終更新日: 2026年2月22日

1. はじめに

「リアルタイム文字起こし」（以下「本拡張機能」）は、ブラウザタブの音声およびマイク音声をリアルタイムで文字起こしするChrome拡張機能です。サイドパネルおよびポップアップウィンドウで文字起こし結果を表示し、テキストの検索・コピー・エクスポート、およびオンデバイス翻訳機能を提供します。本プライバシーポリシーは、本拡張機能がユーザーのデータをどのように取り扱うかを説明します。

2. 収集するデータ

2.1 OpenAI APIキー

本拡張機能は、文字起こし機能を提供するために、ユーザーまたは管理者が提供するOpenAI APIキーを使用します。APIキーは、ユーザーが直接入力するAPIキー（sk-形式）と、管理者が配布するアプリ専用キー（skt-形式）の2種類に対応しています。いずれの形式もブラウザのローカルストレージ内にAES-GCM方式で暗号化して保存されます。アプリ専用キーは拡張機能内で復号され、元のAPIキーとしてOpenAIのAPIへの接続認証にのみ使用されます。

2.2 音声データ

本拡張機能は、ユーザーが明示的に文字起こしを開始した場合に限り、以下の音声データを取得します。

  • ユーザーが共有を許可したブラウザタブの音声（PC音声）
  • ユーザーが有効化した場合のマイク音声

これらの音声データはOpenAIのRealtime Transcription APIにWebSocket（WSS）経由で直接送信されます。音声データは本拡張機能の開発者のサーバーには一切送信されません。

また、ユーザーが「音声を録音する」オプションを有効にした場合、PC音声およびマイク音声を含む録音ファイル（WebM形式）がユーザーの端末にローカル保存されます。録音ファイルは文字起こし停止時に自動的にダウンロードされ、ユーザーの端末内にのみ存在します。録音ファイルが外部に送信されることはありません。

2.3 文字起こし結果

OpenAI APIから返された文字起こしテキストは、ブラウザのローカルストレージ（chrome.storage.local）に保存されます。このデータはユーザーの端末内にのみ存在し、外部には送信されません。

2.4 設定データ

言語設定、文字起こしモデル、区切り方、文字サイズ、フォント、テーマ、マイク使用の有無、マイクノイズリダクション設定などのユーザー設定は、ブラウザのローカルストレージに保存されます。

ユーザーが入力した「用語ヒント（プロンプト）」は、文字起こしの精度向上を目的としてOpenAI APIのセッション設定の一部として送信されます。

拡張機能の内部動作に必要な一時的な状態情報（マイクタブの識別子、ポップアップウィンドウの状態など）は、ブラウザのセッションストレージに保存され、ブラウザを閉じると自動的に削除されます。

2.5 翻訳データ

本拡張機能は、Chromeブラウザに組み込まれたTranslator API（Chrome 138以降）を使用して、文字起こし結果のテキストを翻訳します。翻訳処理はすべてユーザーの端末上のオンデバイスAIモデルによって実行され、翻訳対象のテキストが外部サーバーに送信されることはありません。翻訳結果は文字起こし結果と同様にブラウザのローカルストレージに保存されます。

3. 収集しないデータ

本拡張機能は以下のデータを一切収集しません。

  • 個人を特定できる情報（氏名、メールアドレス、住所など）
  • ウェブ閲覧履歴、アクセスしたURL
  • ウェブページのコンテンツ
  • Cookie、位置情報、財務情報
  • 匿名の使用統計やアナリティクスデータ
  • 翻訳処理に伴う外部へのテキスト送信（翻訳はすべて端末内で完結します）

4. データの使用目的

収集したデータは以下の目的にのみ使用されます。

  • OpenAI APIキー: OpenAI Realtime Transcription APIへの接続認証
  • 音声データ: OpenAI APIによるリアルタイム文字起こし処理
  • 録音ファイル: ユーザーが有効にした場合の音声記録のローカル保存
  • 文字起こし結果: ユーザーへの表示、検索、コピー、エクスポート機能の提供
  • 用語ヒント: OpenAI APIでの文字起こし精度の向上
  • 設定データ: ユーザーの設定の保持・復元
  • 翻訳データ: 文字起こし結果のオンデバイス翻訳表示

5. データの共有・第三者への提供

音声データおよびユーザーが設定した用語ヒントは、OpenAI, Inc.のAPIに直接送信されます。OpenAIによるデータの取り扱いについては、OpenAIのプライバシーポリシーをご確認ください。

上記を除き、本拡張機能の開発者はユーザーのデータを第三者に共有、販売、転送することは一切ありません。

6. データの保存とセキュリティ

  • OpenAI APIキーはAES-GCM（256ビット）で暗号化してブラウザのローカルストレージに保存されます。
  • アプリ専用キー（skt-形式）は、拡張機能内でAES-GCMにより復号され、復号後のAPIキーはメモリ上でのみ使用され永続的に平文で保存されることはありません。
  • 音声データはWebSocket Secure（WSS）で暗号化して送信されます。
  • 文字起こし結果および設定データはブラウザのローカルストレージにのみ保存され、外部送信されません。
  • 録音ファイルはユーザーの端末にのみ保存され、外部に送信されません。
  • すべてのデータはユーザーの端末内にのみ存在します。

7. データの削除

ユーザーはいつでも以下の方法でデータを削除できます。

  • 拡張機能内の「クリア」ボタンで文字起こし履歴を削除
  • 設定画面でAPIキーを削除
  • 録音ファイルはユーザーの端末に保存されるため、通常のファイル操作で削除できます
  • 拡張機能をアンインストールすることで、拡張機能が保存したすべてのデータが削除されます（端末にダウンロードされた録音ファイルを除く）

8. Chrome Web Store Limited Use Policy への準拠

本拡張機能は、Chrome Web Store User Data Policyおよび Limited Use Requirements に準拠しています。取得したユーザーデータは、本拡張機能の単一の目的（リアルタイム文字起こし）を提供・改善するためにのみ使用されます。ユーザーデータを広告目的で使用すること、第三者に販売すること、信用力の判定に使用することはありません。

9. 子どものプライバシー

本拡張機能は13歳未満の子どもを対象としたものではなく、13歳未満の子どもから意図的に個人情報を収集することはありません。

10. 本ポリシーの変更

本プライバシーポリシーは必要に応じて更新されることがあります。重要な変更がある場合は、本ページの最終更新日を更新します。

11. お問い合わせ

本プライバシーポリシーに関するご質問は、Chrome Web Storeの拡張機能サポートページよりお問い合わせください。

Privacy Policy — Real-time Transcription

Last updated: February 22, 2026

1. Introduction

"Real-time Transcription" (hereinafter "this extension") is a Chrome extension that transcribes audio from browser tabs and microphone input in real time. It displays transcription results in a side panel and a popup window, and provides text search, copy, export, and on-device translation features. This Privacy Policy explains how this extension handles user data.

2. Data We Collect

2.1 OpenAI API Key

This extension uses an OpenAI API key provided by the user or an administrator to deliver transcription functionality. Two key formats are supported: standard API keys entered directly by the user (sk- format) and app-bound keys distributed by an administrator (skt- format). Both formats are encrypted using AES-GCM and stored in the browser's local storage. App-bound keys are decrypted within the extension and used solely for authenticating connections to the OpenAI API.

2.2 Audio Data

This extension captures the following audio data only when the user explicitly initiates transcription:

  • Audio from the browser tab that the user has authorized for sharing (PC audio)
  • Microphone audio, if enabled by the user

Audio data is sent directly to the OpenAI Realtime Transcription API via WebSocket Secure (WSS). Audio data is never sent to the extension developer's servers.

Additionally, if the user enables the "Record audio" option, a recording file (WebM format) containing PC audio and microphone audio is saved locally on the user's device. The recording file is automatically downloaded when transcription stops and exists only on the user's device. Recording files are never transmitted externally.

2.3 Transcription Results

Transcription text returned from the OpenAI API is stored in the browser's local storage (chrome.storage.local). This data exists only on the user's device and is not transmitted externally.

2.4 Settings Data

User preferences such as language, transcription model, segmentation mode, font size, font family, theme, microphone usage, and microphone noise reduction settings are stored in the browser's local storage.

The "terminology hints (prompt)" entered by the user are sent to the OpenAI API as part of the session configuration to improve transcription accuracy.

Temporary state information required for the extension's internal operation (such as microphone tab identifiers and popup window state) is stored in the browser's session storage and is automatically deleted when the browser is closed.

2.5 Translation Data

This extension uses the Translator API built into Chrome (Chrome 138 and later) to translate transcription text. All translation processing is performed entirely on the user's device using an on-device AI model, and the text being translated is never sent to any external server. Translation results are stored in the browser's local storage in the same manner as transcription results.

3. Data We Do Not Collect

This extension does not collect any of the following:

  • Personally identifiable information (name, email address, physical address, etc.)
  • Web browsing history or visited URLs
  • Web page content
  • Cookies, location data, or financial information
  • Anonymous usage statistics or analytics data
  • External transmission of text for translation processing (all translation is performed entirely on-device)

4. How We Use Data

Collected data is used solely for the following purposes:

  • OpenAI API Key: Authentication for connecting to the OpenAI Realtime Transcription API
  • Audio Data: Real-time transcription processing via the OpenAI API
  • Recording Files: Local storage of audio recordings when enabled by the user
  • Transcription Results: Display, search, copy, and export functionality for users
  • Terminology Hints: Improving transcription accuracy via the OpenAI API
  • Settings Data: Persisting and restoring user preferences
  • Translation Data: On-device translation and display of transcription results

5. Data Sharing and Third-Party Disclosure

Audio data and user-configured terminology hints are sent directly to OpenAI, Inc.'s API. For information on how OpenAI handles data, please refer to OpenAI's Privacy Policy.

Other than the above, the extension developer does not share, sell, or transfer user data to any third parties.

6. Data Storage and Security

  • The OpenAI API key is encrypted with AES-GCM (256-bit) and stored in the browser's local storage.
  • App-bound keys (skt- format) are decrypted via AES-GCM within the extension; the decrypted API key is used only in memory and is never persistently stored in plaintext.
  • Audio data is transmitted using WebSocket Secure (WSS) encryption.
  • Transcription results and settings are stored only in the browser's local storage and are not transmitted externally.
  • Recording files are stored only on the user's device and are never transmitted externally.
  • All data resides solely on the user's device.

7. Data Deletion

Users can delete their data at any time through the following methods:

  • Use the "Clear" button within the extension to delete transcription history
  • Remove the API key from the settings
  • Recording files are saved on the user's device and can be deleted through standard file management
  • Uninstalling the extension deletes all data stored by the extension (excluding recording files already downloaded to the device)

8. Compliance with Chrome Web Store Limited Use Policy

This extension complies with the Chrome Web Store User Data Policy and Limited Use Requirements. User data obtained is used solely to provide and improve the single purpose of this extension (real-time transcription). User data is not used for advertising purposes, sold to third parties, or used for creditworthiness determinations.

9. Children's Privacy

This extension is not intended for children under the age of 13, and we do not knowingly collect personal information from children under 13.

10. Changes to This Policy

This Privacy Policy may be updated as necessary. If significant changes are made, the "Last updated" date at the top of this page will be revised.

11. Contact

For questions about this Privacy Policy, please use the support page on the Chrome Web Store extension listing.
